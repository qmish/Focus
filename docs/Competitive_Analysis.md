## 1. Обзор Focus Messenger

Focus — корпоративный мессенджер на базе open-source стека (Go, React, Jitsi Meet, Keycloak), предназначенный для on-premise развёртывания в корпоративной инфраструктуре.

### Текущие возможности (реализовано)

| Категория | Функциональность |
|-----------|-----------------|
| **Чат** | Личные и групповые комнаты, обмен сообщениями в реальном времени (WebSocket), отправка файлов |
| **Видеоконференции** | Jitsi Meet (кастомизированный форк) — неограниченное число участников, брендирование UI |
| **Авторизация** | Локальная + Keycloak SSO (OIDC/PKCE), роли user/moderator/admin/service, RBAC + ABAC |
| **Боты** | Встроенный bot-engine, команды /create, /schedule, /status, /joke, /dice, /motivation |
| **Вебхуки** | Inbound/outbound с подписью, retry/backoff, dead-letter, админ-мониторинг |
| **Календарь** | Интеграция с MS Exchange (события, уведомления об отмене) |
| **Админ-панель** | Пользователи, статистика, баны, конференции, аудит webhook/bot, аудит auth/calendar |
| **Десктоп** | Windows (NSIS installer) + macOS (DMG: aarch64 + x64) на Tauri 2 |
| **Уведомления** | Нативные push-уведомления (десктоп), WebSocket real-time |
| **Инфраструктура** | Kubernetes (stage/prod), Docker Compose (dev), CI/CD (GitHub Actions), Prometheus + Grafana |
| **Безопасность** | JWT с ротацией, WS auth + room-level access, origin allowlist, CSP, TLS, security gates в CI |

### Запланировано / в процессе

- Полная интеграция AD (Azure AD / on-prem ADFS) через Keycloak
- SCIM/LDAP sync
- MS Exchange доступ через Graph API (delegated/OBO)
- Load-тестирование в целевых профилях
- UAT с пилотной группой
- Security hardening (закрытие high/critical findings)
- Мобильные приложения (не начато)

## 3. SWOT-анализ Focus Messenger

### Сильные стороны (Strengths)

1. **Полный контроль** — on-premise, open-source стек, нет vendor lock-in
2. **Зрелая ВКС** — Jitsi Meet (проверенное решение, без ограничений на участников)
3. **Продвинутая auth** — Keycloak SSO + RBAC + ABAC (уникально для этого сегмента)
4. **Вебхуки production-grade** — retry/backoff, dead-letter, подпись, админ-мониторинг
5. **Полноценная админка** — аудит, мониторинг, баны, конференции
6. **Десктоп-клиенты** — Windows + macOS (Tauri 2, нативные уведомления, трей)
7. **CI/CD и DevOps** — Kubernetes, Docker Compose, GitHub Actions, security/quality gates
8. **API-first** — Swagger/OpenAPI, документированный REST API
9. **Бот-движок** — встроенный, расширяемый, интегрированный с Exchange/Jitsi

### Слабые стороны (Weaknesses)

1. **Базовый чат** — нет тредов, каналов, реакций, @упоминаний, закрепления, редактирования
2. **Нет мобильных приложений** — критично для корпоративного использования
3. **Нет E2E-шифрования** — конкуренты (eXpress, Rocket.Chat, Element) предлагают из коробки
4. **Нет записи ВКС** — большинство конкурентов предлагают запись конференций
5. **Нет DLP** — контроль утечек данных, блокировка скриншотов, ограничение пересылки
6. **Нет сертификации** — ФСТЭК, ФСБ (критично для госструктур)
7. **Малая команда** — скорость разработки ограничена
8. **Нет offline-режима** — нет работы без интернета

### Возможности (Opportunities)

1. **Мобильные приложения** — Tauri 2 Mobile или React Native / Flutter
2. **AI-функции** — транскрипция, умный поиск, автопротоколирование (тренд рынка)
3. **Marketplace ботов/интеграций** — расширяемость без изменения ядра
4. **Сертификация** — выход на рынок госструктур и регулируемых отраслей
5. **Треды + реакции** — быстрый рост UX без переписывания архитектуры
6. **Запись ВКС** — Jitsi поддерживает Jibri (server-side recording)
7. **Импорт из Slack/Teams** — миграция корпоративных клиентов
8. **Linux-клиент** — Tauri 2 поддерживает Linux из коробки

### Угрозы (Threats)

1. **Конкуренция** — рынок российских корп. мессенджеров насыщен (Compass, eXpress, МТС Линк, Pachca)
2. **Скорость развития конкурентов** — AI-функции появляются у YuChat, DION, MAX
3. **Регуляторные требования** — ужесточение требований к локализации данных и сертификации
4. **Jitsi upstream** — сложность сопровождения форка при частых обновлениях

## 7. Итоговое заключение

**Focus Messenger** занимает нишу **on-premise корпоративного мессенджера с сильной инфраструктурой и безопасностью**, но с незрелым UX чата. Конкурентные преимущества:

- **Keycloak SSO + RBAC + ABAC** — уникально для сегмента
- **Jitsi Meet** — зрелое ВКС без ограничений
- **Production-grade вебхуки** — лучше большинства конкурентов
- **Полный контроль** — on-premise, свой код, нет vendor lock-in
- **Десктоп-клиенты** — Windows + macOS уже в релизе

Главные разрывы для закрытия: **UX чата** (треды, реакции, упоминания), **мобильные приложения**, **запись ВКС**.

При реализации краткосрочной стратегии (1-3 мес.) Focus выйдет на паритет с Compass и Mattermost по функциональности чата, сохраняя превосходство в auth/webhooks/admin-инструментах и ВКС.
