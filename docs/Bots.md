# Bots

**Версия:** 2.0  
**Дата:** 25 марта 2026 г.

---

## 1. Назначение

`BotEngine` обрабатывает slash-команды в комнатах и публикует ответы как обычные сообщения от системного пользователя бота. Начиная с версии 2.0, движок поддерживает расширенные типы обработчиков, запланированные сообщения, индивидуальные идентичности ботов и библиотеку шаблонов.

---

## 2. Компоненты

- `internal/bots/bots.go`
  - парсинг команд
  - rate limiting (глобальный и per-command)
  - вызов handler команд (встроенные + пользовательские)
  - сохранение и доставка ответа
- `internal/bots/scheduler.go`
  - периодическая проверка напоминаний
  - запуск запланированных сообщений по cron-расписанию
- `bot_command_events`
  - аудит и наблюдаемость выполнения команд
- `bot_reminders`
  - хранение пользовательских напоминаний (`/remind`)
- `bot_dialog_states`
  - состояние многошаговых диалогов (будущее)
- зависимости:
  - `RoomRepository`
  - `UserRepository`
  - `MessageRepository`
  - `WebSocket hub`
  - `CalendarService` (для `/schedule`)
  - `BotSettingsProvider` (загрузка конфигурации из БД)
  - `BotReminderStore` (напоминания)

## 4. Типы обработчиков пользовательских команд

| Тип | Описание | Пример `description` |
|-----|----------|----------------------|
| `static-reply` | Фиксированный текстовый ответ | `Pong!` |
| `template` | Ответ с подстановкой переменных | `Привет, {{user_name}}! Сегодня {{date}}.` |
| `random` | Случайный выбор из списка ответов | `ответ1\|\|ответ2\|\|ответ3` |
| `alias` | Перенаправление на другую команду | `help` (алиас на `/help`) |
| `webhook` | HTTP POST на внешний URL | URL в поле `webhook_url` |

### Шаблонные переменные (handler: template)

| Переменная | Описание |
|------------|----------|
| `{{user_name}}` | Имя пользователя |
| `{{user_email}}` | Email пользователя |
| `{{room_name}}` | Название комнаты |
| `{{room_type}}` | Тип комнаты (public/private/meeting) |
| `{{date}}` | Текущая дата (DD.MM.YYYY) |
| `{{time}}` | Текущее время (HH:MM) |
| `{{args}}` | Аргументы команды |
| `{{user_id}}` | UUID пользователя |
| `{{room_id}}` | UUID комнаты |

### Webhook формат запроса

```json
POST <webhook_url>
Content-Type: application/json
X-Focus-Bot-Command: <command>

{
  "command": "deploy",
  "args": "production",
  "user_id": "uuid",
  "room_id": "uuid",
  "timestamp": "2026-03-25T10:00:00Z"
}
```

## 5. Запланированные сообщения

Каждый `BotSetting` может содержать поле `schedule_json` — массив cron-записей:

```json
[
  {"cron": "0 10 * * 1-5", "room_id": "uuid", "message": "Доброе утро!"}
]
```

Формат cron: `минута час день месяц день_недели` (0-6, 0=воскресенье).

`BotScheduler` проверяет расписания каждые 15 секунд и отправляет сообщения в указанные комнаты.

---

## 6. Напоминания

Команда `/remind <время> <сообщение>` создаёт запись в таблице `bot_reminders`. Планировщик проверяет и отправляет напоминания по наступлению `fire_at`.

Поддерживаемые форматы времени: `30s`, `5m`, `2h`, `1d`.

---

## 7. Поток выполнения

1. Сообщение начинается с `/`.
2. Проверка: команда не отключена (`disabled`).
3. Проверка: комната в списке разрешённых (если задан `allowed_rooms`).
4. Проверка доступа пользователя к комнате.
5. Проверка per-command rate-limit (или глобального).
6. Маршрутизация к обработчику:
   - Встроенный handler → прямой вызов.
   - `static-reply` → возврат `description`.
   - `template` → подстановка переменных.
   - `random` → случайный выбор из `description`.
   - `alias` → перенаправление на целевую команду.
   - `webhook` → HTTP POST на внешний URL.
7. Сохранение bot message в БД.
8. Broadcast сообщения в WebSocket.
9. Запись события выполнения в `bot_command_events`.

## 8. Модель данных BotSetting

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | UUID | Первичный ключ |
| `name` | VARCHAR(128) | Уникальное имя бота |
| `description` | TEXT | Описание бота |
| `is_enabled` | BOOLEAN | Включен/выключен |
| `rate_limit_ms` | INT | Глобальный rate limit (мс) |
| `allowed_rooms` | TEXT[] | Массив UUID разрешённых комнат |
| `commands_json` | TEXT | JSON массив BotCommand |
| `schedule_json` | TEXT | JSON массив BotScheduleEntry |
| `bot_user_id` | UUID | Индивидуальный пользователь бота |
| `avatar_url` | VARCHAR(512) | URL аватара бота |

### BotCommand JSON

```json
{
  "command": "/ping",
  "handler": "static-reply",
  "description": "Pong!",
  "is_active": true,
  "webhook_url": "",
  "rate_limit_ms": 0
}
```

## 11. Ошибки и наблюдаемость

- Ошибки команд возвращаются как человекочитаемые bot-ответы.
- Ошибки доставки/сохранения не ломают основной message flow.
- Все критичные сбои фиксируются в `bot_command_events` для admin-наблюдаемости.
- Per-command статистика доступна через `/admin/bots/command-stats`.
- Полная история с фильтрацией через `/admin/bots/command-history`.
- Экспорт истории в CSV из админ-панели.
